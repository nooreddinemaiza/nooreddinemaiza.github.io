Framework maison

NM Base Framework

Micro-framework PHP entièrement orienté objet, développé de zéro. Il fournit un routeur interne, une architecture MVC, un autoloader PSR-4 et une couche d'accès base de données — servant de socle à tous les projets PHP du portfolio.

PHP OOP MVC Router PSR-4 PDO
Type Framework PHP interne
Paradigme OOP — MVC
Version PHP 8.0 +
Utilisé dans Tous les projets PHP

Structure du framework

nm-base-framework/ ├── core/ ← Noyau du framework │ ├── Router.php ← Routeur interne (GET/POST/PUT/DELETE) │ ├── Controller.php ← Contrôleur de base (render, redirect…) │ ├── Model.php ← Modèle de base (query builder léger) │ ├── Database.php ← Singleton PDO │ ├── Request.php ← Encapsulation $_GET/$_POST/$_FILES │ ├── Response.php ← JSON / redirect / status codes │ ├── Session.php ← Gestion sessions & flash messages │ ├── View.php ← Moteur de templates PHP natif │ ├── Middleware.php ← Interface middleware (auth, csrf…) │ └── Autoloader.php ← PSR-4 sans Composer (optionnel) │ ├── app/ ← Code applicatif (à surcharger) │ ├── Controllers/ │ ├── Models/ │ ├── Views/ │ │ ├── layouts/ ← Templates de base (header, footer) │ │ └── partials/ ← Composants réutilisables │ └── Middleware/ │ ├── public/ │ ├── index.php ← Point d'entrée unique (Front Controller) │ ├── .htaccess ← Réécriture URL Apache │ ├── css/ js/ images/ │ └── config/ ├── app.php ← Config générale (env, debug, timezone) ├── database.php ← DSN, host, user, pass, dbname └── routes.php ← Déclaration de toutes les routes

Cycle de vie d'une requête

01
Front Controller
Toutes les requêtes HTTP sont redirigées vers public/index.php grâce au .htaccess. C'est le seul point d'entrée de l'application.
02
Bootstrap & Autoloader
L'autoloader PSR-4 est enregistré, la configuration est chargée (config/app.php, config/database.php), la session est démarrée.
03
Routeur
Le Router compare l'URI et la méthode HTTP aux routes définies dans config/routes.php et résout le contrôleur + l'action cibles.
04
Middlewares
La pile de middlewares associée à la route est exécutée en séquence (auth, CSRF, rate-limiting…). Un middleware peut court-circuiter la chaîne et renvoyer directement une réponse.
05
Contrôleur → Modèle → Vue
Le contrôleur instancie les modèles nécessaires, traite la logique métier, puis appelle $this->render('vue', $data) qui injecte les données dans le template PHP natif.
06
Réponse HTTP
La classe Response envoie les headers appropriés (Content-Type, status code) et renvoie le HTML, le JSON ou effectue une redirection.

Exemples de code

Déclaration des routes dans config/routes.php :

config/routes.php // Déclaration des routes de l'application use Core\Router; Router::get('/', 'HomeController@index'); Router::get('/users', 'UserController@index', ['auth']); Router::get('/users/{id}', 'UserController@show', ['auth']); Router::post('/users', 'UserController@store', ['auth', 'csrf']); Router::put('/users/{id}', 'UserController@update', ['auth', 'csrf']); Router::delete('/users/{id}', 'UserController@destroy', ['auth']);

Exemple d'un contrôleur héritant de la base :

app/Controllers/UserController.php namespace App\Controllers; use Core\Controller; use App\Models\User; class UserController extends Controller { public function index(): void { $users = User::all(); // Rend la vue resources/views/users/index.php $this->render('users/index', compact('users')); } public function store(): void { $data = $this->request->only(['username', 'password', 'email']); User::create($data); $this->session->flash('success', 'Utilisateur créé.'); $this->redirect('/users'); } }

Modèle de base avec query builder léger :

app/Models/User.php namespace App\Models; use Core\Model; class User extends Model { protected static string $table = 'users'; // Core\Model fournit : all(), find(), where(), create(), update(), delete() // Méthode personnalisée : public static function findByEmail(string $email): ?self { return static::where('email', $email)->first(); } }

Ce que fournit le framework

Routeur interne supportant GET, POST, PUT, DELETE avec paramètres dynamiques dans l'URI ({id}, {slug}).
Système de middlewares en pipeline : chaque route peut embarquer une liste de middlewares exécutés avant l'action du contrôleur.
Moteur de vues en PHP natif avec layouts et partials — pas de langage de template externe, juste du PHP.
Couche Model avec méthodes statiques (all(), find(), where(), create(), update(), delete()) via PDO préparé.
Singleton PDO partagé dans toute l'application, configurable par environnement.
Gestion des sessions avec flash messages (succès, erreurs) accessibles dans les vues.
Classe Request pour accéder proprement aux données entrantes (only(), get(), post(), file()).
Autoloader PSR-4 intégré — utilisable avec ou sans Composer selon le contexte de déploiement.
